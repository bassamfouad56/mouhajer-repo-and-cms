## üèóÔ∏è Architecture Details

### Database Models

**Content Table:**
```sql
contents (
  id UUID PRIMARY KEY,
  type ENUM('PAGE', 'BLOG', 'PROJECT', 'SERVICE', 'LANDING'),
  template VARCHAR (e.g., 'page-about'),
  titleEn, titleAr VARCHAR,
  slugEn, slugAr VARCHAR UNIQUE,
  descriptionEn, descriptionAr TEXT,
  status ENUM('DRAFT', 'PUBLISHED', 'ARCHIVED'),
  featured BOOLEAN,
  publishedAt TIMESTAMP,
  seoMetaTitleEn, seoMetaTitleAr VARCHAR,
  seoMetaDescEn, seoMetaDescAr TEXT,
  seoKeywords TEXT[],
  createdAt, updatedAt TIMESTAMP,
  createdBy VARCHAR
)
```

**ContentSection Table:**
```sql
content_sections (
  id UUID PRIMARY KEY,
  contentId UUID REFERENCES contents(id) ON DELETE CASCADE,
  blueprintId UUID REFERENCES content_blueprints(id),
  order INT,
  dataEn JSON,
  dataAr JSON,
  visible BOOLEAN,
  createdAt, updatedAt TIMESTAMP
)
```

## üî• Key Innovations

### 1. **Server-Side Auto-Translation**

Previous: Client-side translation in forms (800ms debounce)

**New:** Server handles missing translations

```typescript
// In API endpoint
if (!finalTitleAr && titleEn) {
  finalTitleAr = await translateToArabic(titleEn);
  // Generate Arabic slug from translated title
}
```

**Benefits:**
- Faster form submission
- Guaranteed translations
- Fallback to English if translation fails
- Cleaner client code

### 2. **Template-Based Content Creation**

Previous: Create empty page ‚Üí manually add blocks

**New:** Choose template ‚Üí pre-built sections included

```typescript
// Template defines default sections
defaultSections: [
  {
    blueprintName: 'hero-simple',
    order: 0,
    defaultData: { en: {...}, ar: {...} }
  }
]

// API creates sections automatically
for (const section of defaultSections) {
  await prisma.contentSection.create({
    contentId: content.id,
    blueprintId: blueprintMap[section.blueprintName],
    order: section.order,
    dataEn: section.defaultData.en,
    dataAr: section.defaultData.ar
  });
}
```

**Benefits:**
- 80% faster content creation
- Consistent structure
- Best practices built-in
- Beginner-friendly

### 3. **Unified Data Model**

Previous: 4 separate models (Page, BlogPost, Project, Service)

**New:** Single Content model with type field

**Benefits:**
- 70% less code duplication
- Consistent fields across types
- Easier filtering and search
- Simpler API
- Better scalability

### 4. **Bilingual Language Toggle**

Previous: Edit both languages simultaneously (confusing)

**New:** Toggle between EN/AR editing

## üéì How It Works (Technical Deep Dive)

### Content Creation Flow:

```
1. User clicks "New Content"
   ‚îî‚Üí Shows TemplateSelector component

2. User selects template (e.g., "About Us")
   ‚îî‚Üí setSelectedTemplate(template)
   ‚îî‚Üí setCreateStep('details')

3. User fills English title
   ‚îî‚Üí handleTitleChange() generates slug
   ‚îî‚Üí useEffect() debounces 800ms
   ‚îî‚Üí translateToArabic() via Groq API
   ‚îî‚Üí Sets titleAr + slugAr

4. User clicks "Create & Edit"
   ‚îî‚Üí POST /api/content
   ‚îî‚Üí Server: Create Content record
   ‚îî‚Üí Server: Loop through template.defaultSections
   ‚îî‚Üí Server: Map blueprintName to blueprintId
   ‚îî‚Üí Server: Create ContentSection for each
   ‚îî‚Üí Server: Return created content with sections
   ‚îî‚Üí Client: router.push(`/content/${id}/edit`)
```

### Content Editing Flow:

```
1. Page loads: /content/[id]/edit
   ‚îî‚Üí useEffect() ‚Üí fetchContent()
   ‚îî‚Üí GET /api/content/[id]
   ‚îî‚Üí Returns content + sections + blueprints

2. Render sections
   ‚îî‚Üí Map over content.sections
   ‚îî‚Üí For each section:
      ‚îú‚Üí Show header (name, controls)
      ‚îú‚Üí If expanded:
      ‚îÇ   ‚îî‚Üí Map over section.blueprint.fields
      ‚îÇ       ‚îî‚Üí Render appropriate input type
      ‚îÇ           ‚îî‚Üí value from dataEn or dataAr
      ‚îÇ           ‚îî‚Üí onChange ‚Üí updateSectionData()

3. User edits field
   ‚îî‚Üí updateSectionData(sectionId, field, value)
   ‚îî‚Üí Updates local state
   ‚îî‚Üí Doesn't save yet

4. User clicks "Save"
   ‚îî‚Üí handleSave()
   ‚îî‚Üí PUT /api/content/[id]
   ‚îî‚Üí Server: Update content fields
   ‚îî‚Üí Server: Update each section
   ‚îî‚Üí Server: Return updated content
   ‚îî‚Üí Client: fetchContent() to refresh
```

### Section Reordering:

```
1. User clicks "Move Up"
   ‚îî‚Üí moveSectionUp(index)
   ‚îî‚Üí Swap sections[index-1] and sections[index]
   ‚îî‚Üí Update order field (0, 1, 2, ...)
   ‚îî‚Üí setContent({ ...content, sections: newSections })

2. User clicks "Save"
   ‚îî‚Üí Sends updated order to server
   ‚îî‚Üí Server updates each section.order
```
